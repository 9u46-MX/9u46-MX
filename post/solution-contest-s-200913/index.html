A

考虑，若序列不合法，则必然有至少一位不为 111。
故考虑容斥。
合法非负整数指满足 xbitand⁡t=x∧3∣xx \operatorname{bitand} t = x \land 3 | xxbitandt=x∧3∣x 的非负整数 xxx。
设 ccc 为所有合法非负整数的数量，d(x)d(x)d(x) 为 bitand⁡x=0\operatorname{bitand} x = 0bitandx=0 的合法非负整数的数量。
那么，若钦定最终结果一些位为 000，其余数位不考虑，则此时的方案数为 dn(x)d^n(x)dn(x)（xxx 为钦定位置的状压）。
然后简单容斥即可（容斥的原理我也不太懂，直接套偶加奇减的板子即可）。
时间复杂度 O(tlog⁡t)O(t \log t)O(tlogt)，空间复杂度 O(t)O(t)O(t)，期望得分 60pts。
实际得分 0pts。
错误原因：忘记判断钦定为 000 的数位是否在 ttt 中全部为 111（因为这样才有贡献）。

B

Subtask 1

Subtask 1 好水啊。先拿下。
考虑状压。
可以随便怎么搞出一个组合数取模，就可以转化为答案统计的问题。
考虑枚举出选中的点集后，如何统计答案。
考虑将答案转化为虚树中所有边的长度和的二倍减去虚树直径。
考虑使用 ST LCA，O(1)O(1)O(1) 查询。
可以 O(n2)O(n^2)O(n2) 枚举两点求出直径。
考虑这样一个问题：LCA 最低的两个点到她们 LCA 的距离必然会被统计答案。
于是我们暴力找出这两个点，然后统计答案，将其中一个点修改为 LCA，另一个点删掉。
时间复杂度 O(2nn3)O(2^n n^3)O(2nn3)，空间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

Subtask 2

考虑，仅需要求两点的路径长度，直接枚举两个点，然后统计即可。
时间复杂度 O(n2)O(n^2)O(n2)，空间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

Subtask 3

类似 Subtask 2，直接枚举三个点即可。
时间复杂度 O(n3)O(n^3)O(n3)，空间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

Subtask 4

考虑，仅有一种情况，跑一下 Subtask1 中的统计虚树答案即可。
时间复杂度 O(n3)O(n^3)O(n3)，空间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。

至此，期望得分 70pts。
实际得分 0pts。
错误原因：提交时忘记取消 Subtask 判断代码的注释。

C

Subtask 1

直接枚举全排列即可。
时间复杂度 O(n!⋅n)O(n! \cdot n)O(n!⋅n)，空间复杂度 O(n2)O(n^2)O(n2)。

Subtask 3

记不太清子任务编号了。就是给一棵树那个子任务。
考虑，一个行列式对答案有贡献，当且仅当 ∀i∈[1,n],(i,pi)∈E\forall i \in [1, n], (i, p_i) \in E∀i∈[1,n],(i,pi​)∈E。
我们从一个叶节点开始考虑。假设这个节点为 111 号节点，有且仅有一条连向 222 号节点的边。
那么，p1=2p_1 = 2p1​=2。
若 p2≠1p_2 \neq 1p2​​=1，则必然有 ∀i∈[1,n],pi≠1\forall i \in [1, n], p_i \neq 1∀i∈[1,n],pi​​=1，不合法。故 p2=1p_2 = 1p2​=1。
可删去这两个点。
那么，我们发现，问题可以转化为：给定一棵树，求能否选出 n2\frac n 22n​ 条边，使每个点恰好在一条边中。
直接树形 DP 即可（最近恰好出过一个树选链的 idea，然后这段就特别顺利）。
一开始想这样必然输出 111，不可能输出 −1-1−1，但不太会证明。
反正树状数组逆序对不难写，就在 DP 的同时记录了一下选出的另一个点。
时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)，空间复杂度 O(n)O(n)O(n)。

至此，期望得分 20pts。
实际得分 0pts。
错误原因：忘记取模……
